# Schema

Sumire can create a basic table from entity metadata on SQLite, MySQL, and PostgreSQL.

## Create a Table

```php
$database->createTable(User::class);
```

The method signature is:

```php
public function createTable(string $entityClass, bool $ifNotExists = false): void
```

The table name comes from `#[Table]`, and column names come from `#[Id]` and `#[Column]`. Identifiers are quoted for the active PDO driver.

The database reports an error when the table already exists by default. Allow repeated setup explicitly when needed.

```php
$database->createTable(User::class, ifNotExists: true);
```

## Inferred Types

| PHP mapping | SQLite | MySQL | PostgreSQL |
| --- | --- | --- | --- |
| Generated `#[Id]` with `int` | `INTEGER PRIMARY KEY AUTOINCREMENT` | `BIGINT AUTO_INCREMENT PRIMARY KEY` | `BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY` |
| `int` or int-backed enum | `INTEGER` | `BIGINT` | `BIGINT` |
| `float` | `REAL` | `DOUBLE` | `DOUBLE PRECISION` |
| `bool` | `INTEGER` | `BOOLEAN` | `BOOLEAN` |
| `string` or string-backed enum | `TEXT` | `VARCHAR(255)` | `VARCHAR(255)` |
| `DateTimeInterface` | `TEXT` | `DATETIME` | `TIMESTAMP WITHOUT TIME ZONE` |
| `ColumnType::Json` | `TEXT` | `JSON` | `JSONB` |

`ColumnType::DateTime` uses the same DateTime row even when conversion is selected explicitly.

## Nullability and IDs

Regular columns are `NOT NULL` unless `#[Column(nullable: true)]` is set.

```php
#[Column(nullable: true)]
private ?string $displayName = null;
```

`#[Id(generated: false)]` creates a regular primary key using the property's inferred type. Generated IDs must use `int` or `?int`.

## Unsupported Types

Schema creation throws `Sumire\Exception\MappingException` when a property type cannot be inferred. Arrays and objects must use an explicit supported `ColumnType`, such as JSON.

```php
/** @var array<string, mixed> */
#[Column(type: ColumnType::Json)]
private array $metadata;
```

## Scope

`createTable()` creates columns and a primary key. It does not currently create:

- indexes
- unique constraints
- foreign keys or relationships
- database defaults
- schema migrations or alterations

Use a migration tool for versioned production schema changes. The helper is most useful for examples, tests, prototypes, and small applications with simple tables.
